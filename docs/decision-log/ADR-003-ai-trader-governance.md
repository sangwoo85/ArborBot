# ADR-003: AI 트레이더 거버넌스

- 상태: Accepted
- 일자: 2026-06-25
- 관련: [AI_TRADER_POLICY.md](../../AI_TRADER_POLICY.md), [STRATEGY_GOVERNANCE.md](../../STRATEGY_GOVERNANCE.md), [ADR-002](ADR-002-order-safety.md)

## 맥락

AI가 시장 국면을 판단하고 전략을 선택해 종목별 신호를 만든다. AI 판단은 확률적이고
오류·드리프트 가능성이 있으므로, AI의 권한과 산출물을 명확히 제약해야 한다.

## 결정

1. **산출물 제한**: AI의 유일한 산출물은 `TradingSignal`이다. 주문/취소 권한은 없다.
2. **승인된 전략 풀 내 선택**: AI는 거버넌스를 통과한 전략 중에서만 선택한다.
   임의의 새 전략을 즉흥 생성·실행하지 않는다([STRATEGY_GOVERNANCE.md](../../STRATEGY_GOVERNANCE.md)).
3. **신뢰도·만료 강제**: 모든 신호는 `confidenceScore`와 `validUntil`을 가진다.
   최소 신뢰도 미만 또는 만료 신호는 주문 후보가 되지 못한다.
4. **권장치는 비강제**: `recommendedPositionSize`는 권장값일 뿐, 한도가 항상 우선한다.
5. **재현성·추적성**: 신호의 `rationale`와 입력 요약(`signal_input_snapshot`)을 저장한다.
   (비밀·개인정보는 제외)
6. **성과 기반 자기 제한**: 성과가 악화된 전략은 비활성화/비중 축소하고, 필요 시
   Strategy Kill Switch로 차단한다.
7. **포트폴리오 우선**: 개별 종목 기대수익보다 전체 위험을 우선 고려한다.

## 근거

- AI 권한을 "신호 생성"으로 한정하면 오판의 영향이 항상 위험 검증 뒤에서 걸러진다.
- 신뢰도/만료는 stale·저품질 신호로 인한 주문을 줄인다.
- 입력 요약 저장은 사후 분석·전략 개선·디버깅의 기반이 된다.

## 대안과 기각 사유

- **AI에 비중·실행 재량 부여**: 한도 우회·과최적화 위험 → 기각.
- **신호 무기한 유효**: 시장 변화에 취약 → 기각, 만료 강제.

## 결과

- AI는 강력하되 **제약된** 의사결정자이며, 안전은 risk-engine이 보장한다.
- 신호 품질·전략 성과를 지속 모니터링하고 거버넌스로 승격/강등한다.
- 트레이드오프: 즉흥적 기회 포착 제한(의도된 통제).
