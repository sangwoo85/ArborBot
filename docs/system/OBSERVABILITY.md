# OBSERVABILITY — 관측 및 운영 알림

> 메트릭, 로그, 추적, 운영 알림 기준을 정의한다.

관련: [FAILURE_AND_RECOVERY](FAILURE_AND_RECOVERY.md) · [EVENT_FLOW](EVENT_FLOW.md) · [DATA_MODEL](DATA_MODEL.md)

---

## 1. 추적(Tracing)

- 모든 요청/이벤트/로그에 `correlationId` 전파(MDC).
- 신호→주문 추적: `strategySignalId → portfolioDecisionId → orderId`로 드릴다운.
- 분산 추적(선택): OpenTelemetry 호환 트레이스.

---

## 2. 핵심 메트릭

| 범주 | 메트릭 |
| --- | --- |
| 신호 | 생성 수, HOLD 비율, 평균 confidence, 만료/거절 비율 |
| 검증 | ALLOW/PENDING/REJECT 비율, 위반 코드 분포 |
| 주문 | 전송 수, 성공/실패율, 부분체결율, 체결 지연(ms) |
| 손익 | 일일 실현/평가 손익, 한도 대비 사용률(%) |
| 포트폴리오 | 종목/섹터/전략 비중, 현금 보유 비율 |
| 전략 | 전략별 성과 지표, 활성/비활성 수 |
| 인프라 | 외부 API 응답시간/오류율, 서킷브레이커 상태, 큐 적체/DLQ |
| 데이터 | 피드 지연, 결측률, 품질 플래그 |
| 안전 | Kill Switch 상태/발동 횟수 |

---

## 3. 로그

- 구조화 로그(JSON), `correlationId`/`orderId` 포함.
- **비밀정보·계좌번호·토큰 마스킹**(기존 보안 정책 준수).
- 감사 로그는 일반 로그와 분리 저장(append-only).

---

## 4. 운영 알림 기준

다음 이벤트는 **즉시 알림**(채널: 운영자에게 push/메신저 등) 대상이다.

| # | 알림 트리거 | 심각도 | 권장 조치 |
| --- | --- | --- | --- |
| 1 | **주문 실패**(FAILED/REJECTED by broker) | High | 사유 확인, 재동기화 |
| 2 | **API 인증 실패**(토큰 만료/거부) | Critical | 토큰 회전, 주문 차단 점검 |
| 3 | **일일 손실 한도 초과/임계 도달** | Critical | 신규 매수 차단 확인, 모드 강등 검토 |
| 4 | **잔고 불일치** | Critical | 전역 Kill Switch, 재동기화 |
| 5 | **전략 비활성화** | High | 원인 확인, 비중 재배분 |
| 6 | **Kill Switch 활성화** | Critical | 원인·범위 확인, 복구 절차 착수 |
| 7 | **데이터 수집 실패**(피드 중단/지연) | High | 신호 보류 확인, 소스 점검 |
| 8 | 주문 실패율 급증 | High | 서킷브레이커/네트워크 점검 |
| 9 | 부분 체결 장기 미해소 | Medium | 잔량 정책 적용 |
| 10 | DLQ 적체 | Medium | 이벤트 소비 점검 |

알림에는 `correlationId`, 범위, 시각, 권장 조치 링크를 포함한다.

---

## 5. 대시보드 구성(권장)

- **거래 현황**: 신호/주문/체결/실패, 한도 사용률
- **위험**: 손익 한도, 비중 히트맵, Kill Switch 상태
- **전략**: 전략별 성과·활성 상태
- **인프라**: API 응답/오류, 큐/DLQ, 데이터 품질

---

## 6. SLO(예시)

| 항목 | 목표(예시) |
| --- | --- |
| 주문 전송 성공률 | ≥ 99% (장중) |
| 신호→검증 지연 | p95 < 2s |
| 데이터 신선도 | 지연 < 임계(전략별) |
| Kill Switch 발동 후 차단 반영 | < 1s |

> 수치는 예시이며 운영 기준으로 조정한다. 위반 시 알림·자동 보호(보류/Kill)와 연동한다.
