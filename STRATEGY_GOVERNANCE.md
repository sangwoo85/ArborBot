# STRATEGY_GOVERNANCE.md — 전략 거버넌스

본 문서는 전략의 **등록, 백테스트 승인, 페이퍼 트레이딩 검증, 실거래 승격, 전략 중단 기준**을 정의합니다.
목표는 **재현 가능한 검증을 거친 전략만** 실거래에 도달하도록 하는 것입니다.

---

## 1. 전략 생명주기

```
DRAFT → BACKTEST → PAPER → SEMI_AUTO → AUTO(소액) → AUTO(확대)
                                    │
                                 RETIRED (중단/폐기)  ← 어느 단계에서든 강등 가능
```

각 단계 승격은 **정량 기준 충족 + 명시적 승인 기록**이 필요합니다. **단계 건너뛰기 금지.**

---

## 2. 전략 등록 (DRAFT)

전략 등록 시 다음 메타데이터를 `strategy`에 기록합니다.

| 항목 | 설명 |
| --- | --- |
| `strategyId` | 고유 식별자(버전 포함, 예: `momentum-v3`) |
| `description` | 전략 가설/논리 |
| `targetRegime` | 적합 시장 국면(BULL/BEAR/SIDEWAYS/HIGH_VOL/EVENT) |
| `universe` | 적용 종목군/필터 |
| `parameters` | 파라미터 및 기본값 |
| `riskProfile` | 기대 변동성/최대 허용 MDD |
| `owner` | 책임자 |
| `status` | DRAFT |

- 전략 로직과 파라미터는 **버전 관리**한다(파라미터 변경 = 새 버전).
- research/ (Python)에서 구현/연구하고, 재현 가능한 형태로 커밋한다.

---

## 3. 백테스트 승인 (BACKTEST)

### 3.1 요건

- **고정 데이터셋 + 고정 기간**으로 재현 가능해야 한다(동일 입력 → 동일 결과).
- In-sample / Out-of-sample 분리, 가능하면 워크포워드 검증.
- 거래비용·슬리피지·세금을 반영한다.
- 룩어헤드 바이어스/생존 편향을 점검한다.

### 3.2 통과 기준(예시 — 운영값은 별도 관리)

| 지표 | 최소 기준(예시) |
| --- | --- |
| 샤프 비율 | ≥ 1.0 (out-of-sample) |
| 최대 낙폭(MDD) | ≤ riskProfile 한도 |
| 승률/손익비 | 전략 유형에 따라 정의 |
| 표본 거래 수 | 통계적 유의성 확보 가능한 수준 |

> 백테스트 결과(설정·코드 버전·지표)는 보관하며, 미충족 시 DRAFT로 반려한다.

---

## 4. 페이퍼 트레이딩 검증 (PAPER)

- 실시간 신호 + 가상 체결로 **실거래와 동일한 검증 파이프라인**을 통과시킨다(Risk Engine 포함).
- 최소 관찰 기간(예: 4주 이상) 동안 운영한다.
- 백테스트 대비 성과 괴리(슬리피지, 신호 지연, 미체결)를 측정한다.
- 통과 기준(예시): 백테스트 대비 성과 저하가 허용 범위 내, 치명적 운영 오류 없음.

---

## 5. 실거래 승격

승격 순서는 **반드시** 다음을 따릅니다(절대 규칙 10).

```
1. BACKTEST  : 정량 기준 충족
2. PAPER     : 최소 기간 검증 통과
3. SEMI_AUTO : 사람이 모든 주문을 최종 승인. 실 자금, 최소 한도.
4. AUTO(소액): 자동 실행, 한도를 최소값으로 시작.
5. AUTO(확대): 안정성 입증 후 한도 단계적 상향.
```

### 5.1 승격 게이트

| 승격 | 필수 조건 |
| --- | --- |
| → SEMI_AUTO | PAPER 통과 + 운영 책임자 승인 기록 |
| → AUTO(소액) | SEMI_AUTO 일정 기간 무사고 + 거절/실패율 정상 |
| → AUTO 확대 | 누적 손익/위험 지표가 한도 내 + 명시적 승인 |

- 모든 승격/강등은 **감사 로그**에 기록한다(누가, 언제, 근거).
- 한도 상향은 [RISK_POLICY.md](RISK_POLICY.md)의 한도값 변경 절차를 함께 따른다.

---

## 6. 전략 중단/강등 기준 (RETIRED / 강등)

다음 중 하나라도 발생하면 전략을 **즉시 비활성화 또는 비중 축소**한다.

- 실거래 성과가 PAPER/백테스트 대비 심각하게 악화(샤프 급락, MDD 한도 초과)
- 연속 손실 또는 일일 손실 한도 기여가 과도
- 신호 품질 이상(중복/폭주/저신뢰 비율 급증)
- 시장 국면 변화로 전략 가정이 무효화
- 데이터/실행 인프라 결함으로 신뢰 불가

처리:

- AI 트레이더는 해당 전략을 신호 생성에서 제외하거나 비중을 줄인다([AI_TRADER_POLICY.md](AI_TRADER_POLICY.md)).
- 필요 시 **Strategy Kill Switch**로 해당 전략 주문을 차단한다([RISK_POLICY.md](RISK_POLICY.md)).
- 중단 사유와 시점을 감사 로그에 남긴다. 재개는 재검증(BACKTEST/PAPER) 후에만 가능하다.

---

## 7. 정기 리뷰

- 전략별 성과를 주기적으로(예: 주간/월간) 검토하고 비중을 재조정한다.
- 활성 전략 수, 상관관계, 섹터 편중을 점검해 포트폴리오 분산을 유지한다.
